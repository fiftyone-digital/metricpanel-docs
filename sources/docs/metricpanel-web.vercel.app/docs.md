# [MetricPanel documentation](https://metricpanel-web.vercel.app/docs#metricpanel-documentation)

Add MetricPanel to a website or application, verify the data you collect, and turn that activity into goals, funnels, alerts, and attributed revenue.

## [Recommended rollout](https://metricpanel-web.vercel.app/docs#recommended-rollout)

1. Add one production-like website and install either the hosted script or SDK.
2. Confirm a pageview and one intentionally named custom event.
3. Create a goal from that event, then build a two-step funnel around it.
4. Add revenue only after you have chosen a reliable client-to-checkout attribution path.
5. Configure privacy behavior, team access, alerts, and provider health before expanding the rollout.

## [Operator runbooks](https://metricpanel-web.vercel.app/docs#operator-runbooks)

The **Operator runbooks** section in the documentation navigation is deliberately separated from customer setup. Use it for deployment, authentication infrastructure, database operations, observability, preview environments, recovery, self-hosting, troubleshooting, and performance checks. These pages describe running MetricPanel itself; they are not required to install the tracker or use a hosted workspace.
