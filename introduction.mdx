MetricPanel is a privacy-focused web analytics platform that helps you measure what actually drives growth—pageviews, custom events, conversion goals, multi-step funnels, and attributed revenue—without the bloat or compliance headaches of legacy tools. A single 4.4 KB (gzip) hosted script is all you need to get started, and the full JavaScript SDK gives you typed, framework-native control when you need it.

## What MetricPanel Tracks

MetricPanel collects the metrics that matter to product and marketing teams:

- **Pageviews** — automatic on every page load and client-side navigation, including hash-router and History API transitions
- **Custom events** — named actions you define, such as `signup_completed` or `checkout_started`, with optional metadata properties
- **Goals** — conversions matched from event keys, page paths, or a combination of both rules
- **Funnels** — ordered multi-step journeys built from URLs, events, and goals, showing exactly where visitors drop off
- **Revenue attribution** — purchase values tied back to sources, campaigns, landing pages, and funnels via native integrations with Stripe, Polar, LemonSqueezy, and Shopify, or through the revenue events API

## Recommended Rollout Order

Follow this sequence to get reliable data before expanding your instrumentation:

1. **Install the script or SDK** on one production-like website and confirm the script loads without errors.
2. **Confirm a pageview and one custom event** to verify end-to-end data flow before creating any downstream rules.
3. **Create a goal** from that confirmed event, then build a two-step funnel around it.
4. **Add revenue tracking** only after you have identified a reliable client-to-checkout attribution path.
5. **Configure privacy behavior**, team access, alerts, and provider health before expanding the rollout to additional sites.
